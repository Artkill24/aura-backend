## What is AURA?

AURA is a forensic-grade video authenticity verification system targeting B2B clients: **insurance adjusters, law firms, digital forensics teams, and corporate security**.

It goes beyond simple deepfake detection by combining 8 independent analysis layers, generating a cross-layer forensic inference, and producing a legally-structured PDF report with QR-verifiable chain-of-custody.

---

## 8-Layer Detection Pipeline

| Layer | Signal | Weight |
|-------|--------|--------|
| **Metadata** | Encoder tags, GPS, device model, timestamp discrepancy | 7% |
| **Visual AI** | Dual-model deepfake detection (dima806 + umm-maybe) | 5% |
| **Audio Sync** | A/V drift, TTS voice signatures, 50/60Hz spectrum | 7% |
| **Signal Physics** | Block uniformity + edge sharpness (calibrated on FF++) | 25% |
| **Moiré Screen** | LCD pixel grid detection for screen recordings | 8% |
| **PRNU Sensor** | Camera sensor noise fingerprint analysis | 14% |
| **Virtual Cam** | OBS/Deep-Live-Cam injection via sensor noise floor | 13% |
| **rPPG Cardiac** | Biological cardiac signal from green channel FFT (45–150 BPM) | 18% |

**Total: 97%** — remaining 3% reserved for future layers.

## Forensic Inference Engine

AURA doesn't analyze layers in isolation. The cross-layer engine maps convergences to known attack vectors:

| Attack Vector | Trigger Condition |
|---------------|-------------------|
| `REPLAY_ATTACK` | moiré ≥ 0.5 AND vcam ≥ 0.4 |
| `VIRTUAL_CAM_DEEPFAKE` | vcam ≥ 0.5 AND rppg ≥ 0.6 AND moiré < 0.4 |
| `POST_PRODUCTION_DEEPFAKE` | signal ≥ 0.6 AND prnu ≥ 0.5 AND rppg ≥ 0.7 |
| `AI_GENERATED` | visual ≥ 0.5 AND rppg ≥ 0.7 AND signal ≥ 0.5 |
| `AUDIO_MANIPULATION` | audio ≥ 0.5 AND visual ≥ 0.3 |
| `SCREEN_RECORDING_LEGIT` | moiré ≥ 0.5 AND prnu < 0.3 AND composite < 0.4 |

## PDF Report (4 Pages)

Every analysis generates a forensic PDF containing:

1. **Verdict** — composite score, tier, risk color, confidence band
2. **Layer Analysis** — per-layer scores, flags, technical details
3. **Heatmaps** — ELA (Error Level Analysis) + Signal Physics block uniformity map
4. **Forensic Conclusion** — cross-layer attack vector, rPPG summary, recommendations
5. **AI Narrative** — legal-language conclusion generated by Llama 3.3-70B (Groq)
6. **QR Verification** — links to `/verify/{job_id}` endpoint for tamper detection
7. **Chain of Custody** — SHA-256, MD5, model registry, timestamp UTC

## QR Verification

Each report embeds a QR code pointing to:
```
https://artkill24-aura-backend.hf.space/verify/{job_id}?h={sha256[:16]}
```

The `/verify` endpoint returns:
```json
{
  "job_id": "074fff87-...",
  "hash_match": true,
  "verdict": "LIKELY MANIPULATED",
  "composite_score": 0.516,
  "timestamp": "2026-03-13T02:03:57Z"
}
```

If the PDF is tampered with, the hash will not match.

## API Reference

### `POST /analyze`
Upload a video file for forensic analysis.

```bash
curl -X POST https://artkill24-aura-backend.hf.space/analyze \
  -F "file=@video.mp4"
```

**Response:**
```json
{
  "job_id": "uuid",
  "verdict": {
    "label": "LIKELY MANIPULATED",
    "composite_score": 0.516,
    "tier": 5,
    "confidence": "HIGH"
  },
  "visual_score": 0.15,
  "prnu_score": 0.72,
  "rppg_score": 0.85,
  "rppg_quality": "ABSENT",
  "forensic_conclusion": {
    "attack_vector": "POST_PRODUCTION_DEEPFAKE"
  },
  "ai_narrative": "Sulla base dell'analisi...",
  "report_url": "/report/{job_id}",
  "verify_url": "/verify/{job_id}?h={hash}"
}
```

### `GET /report/{job_id}`
Download the forensic PDF report.

### `GET /verify/{job_id}?h={hash}`
Verify report integrity. Returns hash match status and verdict.

### `GET /health`
Health check endpoint.
